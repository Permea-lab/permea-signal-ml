# First Review Wave Checklist v0.1

## Purpose

This checklist operationalizes the first structured review wave for the current BBB-oriented Permea manuscript package.

Use it with `FIRST_REVIEW_WAVE_PLAN_V0_1.md`, `REVIEWER_PACKET_V0_1.md`, `REVIEW_OUTREACH_TEMPLATES_V0_1.md`, `REVIEW_FEEDBACK_LOG_V0_1.md`, and `REVISION_PRIORITY_FRAMEWORK_V0_1.md`.

## Before sending

- [ ] Confirm the package is still framed as draft preprint, not submission-ready.
- [ ] Review `FIRST_REVIEW_WAVE_DRY_RUN_V0_1.md` for packet-routing risks.
- [ ] Confirm no new wet-lab validation, therapeutic, mechanistic, or broad generalization claims were introduced.
- [ ] Confirm the manuscript still describes the work as computational evidence only.
- [ ] Confirm unresolved provenance, attribution, and licensing items remain visible limitations.
- [ ] Confirm the imported-versus-regenerated distinction is preserved.
- [ ] Confirm `PREPRINT_DRAFT_V0_1.md` is the current manuscript draft to send.
- [ ] Review `FIRST_REVIEW_WAVE_OUTREACH_PACKET_V0_1.md` for reviewer-specific routing.

## Packet assembly

- [ ] Assemble the minimum packet:
  - [ ] `PREPRINT_DRAFT_V0_1.md`
  - [ ] `CIRCULATION_GUIDE_V0_1.md`
  - [ ] `REVIEWER_NOTE_V0_1.md`
- [ ] Assemble the extended packet:
  - [ ] `PREPRINT_DRAFT_V0_1.md`
  - [ ] `CIRCULATION_GUIDE_V0_1.md`
  - [ ] `REVIEWER_NOTE_V0_1.md`
  - [ ] `DATASET.md`
  - [ ] `FIRST_EVIDENCE_SUMMARY.md`
  - [ ] `V0_1_EVIDENCE_PACKAGE.md`
  - [ ] `PREPRINT_ASSEMBLY_V0_1.md`
  - [ ] `SUPPLEMENTARY_OUTLINE_V0_1.md`
- [ ] Confirm packet choices against `REVIEWER_PACKET_V0_1.md`.
- [ ] Confirm Reviewer A receives the extended packet by default.

## Reviewer assignment

- [ ] Assign Reviewer A — computational/reproducibility.
- [ ] Assign Reviewer B — biological/skeptical.
- [ ] Decide whether to include Reviewer C — technical clarity.
- [ ] Record reviewer type privately if real names are used outside this repo.
- [ ] Do not add real reviewer names, affiliations, or identities to public docs without explicit intent.

## Outreach

- [ ] Select the relevant template from `REVIEW_OUTREACH_TEMPLATES_V0_1.md`.
- [ ] Use `FIRST_REVIEW_WAVE_OUTREACH_PACKET_V0_1.md` to confirm reviewer-specific packet routing.
- [ ] Customize the message to specify minimum or extended packet.
- [ ] Ask explicitly for claim-boundary feedback.
- [ ] Ask reviewers to flag wording that implies stronger evidence than the package supports.
- [ ] Avoid describing the package as submission-ready.
- [ ] Avoid implying biological validation or solved delivery.

## Feedback intake

- [ ] Enter each substantive comment in `REVIEW_FEEDBACK_LOG_V0_1.md`.
- [ ] Use one row per feedback item.
- [ ] Separate reviewer observation from action decision.
- [ ] Preserve the reviewer concern even if the eventual decision is Reject or Defer.
- [ ] Mark comments that relate to claim boundary, provenance, or factual consistency clearly.

## Triage

- [ ] Assign P0-P4 priority using `REVISION_PRIORITY_FRAMEWORK_V0_1.md`.
- [ ] Mark factual contradictions as P0.
- [ ] Mark overclaim or claim-boundary issues as P1.
- [ ] Mark benchmark, provenance, dataset, or method clarity issues as P2.
- [ ] Mark readability, structure, and formatting improvements as P3.
- [ ] Mark new experiments, new model families, new datasets, or validation requests as P4 unless they expose a current trust-blocking problem.
- [ ] Assign an action decision: Accept, Reject, Defer, Clarify, or Incorporated.

## Revision planning

- [ ] Group accepted P0-P2 items into a file-level revision plan.
- [ ] Fix P0 items before any further circulation.
- [ ] Fix P1 items before external-facing circulation.
- [ ] Resolve or explicitly defer P2 items before preprint submission preparation.
- [ ] Batch P3 items after scientific clarity is stable.
- [ ] Keep P4 items out of the current manuscript revision unless they strengthen the current trust surface without expanding claims.
- [ ] Confirm revisions do not introduce new scientific claims.

## Post-review package refresh

- [ ] Re-read `PREPRINT_DRAFT_V0_1.md` after revisions for claim-boundary drift.
- [ ] Re-check `CIRCULATION_GUIDE_V0_1.md` and `REVIEWER_NOTE_V0_1.md` for consistency.
- [ ] Update `BIORXIV_SUBMISSION_CHECKLIST_V0_1.md` only after first-wave blockers are triaged.
- [ ] Confirm the package still states that it is near-ready for internal or trusted review, not final public submission.
- [ ] Record remaining unresolved questions separately from completed revisions.
