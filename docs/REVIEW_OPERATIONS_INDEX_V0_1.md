# Review Operations Index v0.1

## Purpose

This index points to the documents needed to run the first structured review wave for the current BBB-oriented Permea manuscript package.

## Current status

The package is ready for internal or trusted review. It is not final for public submission.

## Review-operation document map

- `PREPRINT_DRAFT_V0_1.md` — current manuscript draft
- `CIRCULATION_GUIDE_V0_1.md` — reader orientation and evidence boundary
- `REVIEWER_NOTE_V0_1.md` — short note for trusted reviewers
- `REVIEWER_PACKET_V0_1.md` — minimum and extended packet definition
- `REVIEW_OUTREACH_TEMPLATES_V0_1.md` — short outreach messages by reviewer type
- `FIRST_REVIEW_WAVE_PLAN_V0_1.md` — reviewer mix, sequence, expected outputs, and completion criteria
- `FIRST_REVIEW_WAVE_CHECKLIST_V0_1.md` — operational checklist for sending, intake, triage, and refresh
- `REVIEW_FEEDBACK_LOG_V0_1.md` — structured feedback log
- `REVISION_PRIORITY_FRAMEWORK_V0_1.md` — P0-P4 triage and revision rules

## Recommended execution order

1. Read `REVIEWER_PACKET_V0_1.md`.
2. Use `FIRST_REVIEW_WAVE_PLAN_V0_1.md` to assign reviewer types.
3. Use `FIRST_REVIEW_WAVE_CHECKLIST_V0_1.md` during sending and intake.
4. Use `REVIEW_OUTREACH_TEMPLATES_V0_1.md` for concise reviewer asks.
5. Record comments in `REVIEW_FEEDBACK_LOG_V0_1.md`.
6. Triage comments with `REVISION_PRIORITY_FRAMEWORK_V0_1.md`.

## Revision triage path

Use `REVISION_PRIORITY_FRAMEWORK_V0_1.md` to assign P0-P4 priority, separate manuscript revisions from future research tasks, and avoid scope creep.

## Claim-boundary reminder

The current package is computational evidence only. It does not claim wet-lab validation, solved delivery, mechanistic causality, therapeutic efficacy, or broad biological generalization.
