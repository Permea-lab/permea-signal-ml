# Limitations

Permea Signal ML is an early-stage computational evidence repository. Its scope is intentionally narrow.

## Explicit limitations

- no claim of solved delivery
- no claim of wet-lab validation unless such evidence is added later and documented explicitly
- no claim of validated therapeutic success
- no claim of universal permeability prediction
- no claim of broad biological generalization beyond the supported benchmark evidence

## Modeling limitations

- the initial task framing is benchmark-first and simplified
- sequence-derived physicochemical features may capture only part of the relevant signal
- baseline models are useful for calibration, not for final scientific closure
- dataset size, class balance, and source bias may materially affect performance estimates

## Interpretation limitations

- computational ranking is not experimental validation
- benchmark performance is not equivalent to real-world delivery success
- candidate prioritization outputs should be treated as workflow artifacts, not biological proof

## Repository scope limitation

This repository is the first public evidence package in the broader Permea program. It is not the full Permea platform and should not be read as such.

Its purpose is benchmark-oriented and reproducibility-first:

- define a narrow task
- expose the workflow
- measure baseline signal
- support future validation work
