# Reviewer Note v0.1

## Purpose

This note is intended for internal or trusted external reviewers of the current BBB-oriented Permea manuscript package.

## Current manuscript in one paragraph

The current manuscript presents an initial benchmark-oriented computational evidence package for a BBB-oriented peptide benchmark built from sequence-derived physicochemical features. It focuses on baseline reruns using Dummy, Logistic Regression, and Random Forest models, and it interprets the resulting artifacts as support for candidate prioritization prior to experimental validation rather than as biological proof. A central structural feature of the package is the distinction between imported historical artifacts and regenerated current-contract benchmark artifacts, together with a provenance-aware repository structure intended to make the computational evidence more inspectable.

## What feedback is most needed

- claim-boundary clarity
- manuscript coherence
- benchmark legibility
- biological plausibility framing
- figure and table usefulness
- whether the manuscript reads as a serious bounded computational study

## Areas where the package is already strong

- explicit evidence boundaries
- imported-versus-regenerated separation
- reproducible artifact structure
- clear layering between manuscript and support docs
- candid limitations

## Areas where the package is still vulnerable

- provenance closure remains incomplete
- feature surface is limited
- model regime is baseline-oriented
- no experimental validation is included
- formal references are not yet added
- final metadata remain placeholders

## What not to review this as

This package should not be reviewed as a wet-lab validation paper, a therapeutic efficacy paper, a claim of solved delivery, or a broad biological generalization paper. It should instead be reviewed as an initial benchmark-oriented computational evidence package.
